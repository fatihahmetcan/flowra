# API Specification: Customers
**Base Path:** `/api/customers`

## Overview
Management of private and business customers. The `customerCode` is automatically generated by the system upon creation.

## Customer Code Rules
- Format: `C-XXXXXX` (example: `C-000123`)
- Auto-generated by the backend on `POST /`
- Unique across all customers
- Read-only: clients cannot set or change `customerCode`

## Standard Error Format
All error responses use this structure:

```json
{
    "code": "VALIDATION_ERROR",
    "message": "Request validation failed",
    "details": [
        {
            "field": "email",
            "issue": "Invalid email format"
        }
    ]
}
```

---

## Endpoints

### 1. Crate New Customers
Creats a new customer record in the database.

- Method: `Post` 
- URL: `/`
- Authentification: Required (Admin/Office) 
- Validation rules:
  - `type`: required, one of `PRIVATE | BUSINESS`
  - `email`, `phoneNumber`, `streetHouseNumber`, `postalCode`, `city`: required for all customers
  - If `type = BUSINESS`: `companyName` is required
  - If `type = PRIVATE`: `firstName` and `lastName` are required
- Request Body (BUSINESS example): 
```json 
{
    "type": "BUSINESS",
    "companyName": "Example Gardening Ltd",
    "phoneNumber": "+49 171 1234567",
    "email": "office@example-gardening.de",
    "streetHouseNumber": "Main Street 10",
    "postalCode": "60311",
    "city": "Frankfurt"
}
```
- Request Body (PRIVATE example):
```json
{
    "type": "PRIVATE",
    "firstName": "John",
    "lastName": "Doe",
    "phoneNumber": "+49 171 7654321",
    "email": "john.doe@example.com",
    "streetHouseNumber": "Main Street 10",
    "postalCode": "60311",
    "city": "Frankfurt"
}
```
- Success Response: 201 Created
- Response Body (example):
```json
{
    "id": "f7a9d6d0-0f31-4f7d-9a4e-9c6dc7f0e8e2",
    "customerCode": "C-000123",
    "type": "BUSINESS",
    "companyName": "Example Gardening Ltd",
    "firstName": null,
    "lastName": null,
    "phoneNumber": "+49 171 1234567",
    "email": "office@example-gardening.de",
    "streetHouseNumber": "Main Street 10",
    "postalCode": "60311",
    "city": "Frankfurt",
    "createdAt": "2026-03-03T10:30:00.000Z",
    "updatedAt": "2026-03-03T10:30:00.000Z"
}
```
- Error Responses: 
    - 400 Bad Request (Validation failed)
    - 401 Unauthorized (Not authenticated)
    - 403 Forbidden (Missing role: Admin/Office)
    - 409 Conflict (Email already exist)

### 2. List ALl Customers
Retrives a list of all customers, sorted by alphabetical date 

- Method: `GET`
- URL: `/`
- Query Parameters: 
    - `limit` (optional, default: `20`, min: `1`, max: `100`)
    - `offset` (optional, default: `0`, min: `0`)
    - `search` (optional, searches in `customerCode`, `companyName`, `firstName`, `lastName`, `email`)
    - `type` (optional, one of `PRIVATE | BUSINESS`)
    - `sortBy` (optional, one of `createdAt | companyName | lastName`, default: `createdAt`)
    - `sortOrder` (optional, one of `asc | desc`, default: `desc`)
- Default behavior:
    - if no query params are provided, results are sorted by `createdAt desc`
    - pagination uses `limit=20` and `offset=0`
- Success Response 200 OK 
- Response Body (example):
```json
{
    "items": [
        {
            "id": "f7a9d6d0-0f31-4f7d-9a4e-9c6dc7f0e8e2",
            "customerCode": "C-000123",
            "type": "BUSINESS",
            "companyName": "Example Gardening Ltd",
            "firstName": null,
            "lastName": null,
            "phoneNumber": "+49 171 1234567",
            "email": "office@example-gardening.de",
            "streetHouseNumber": "Main Street 10",
            "postalCode": "60311",
            "city": "Frankfurt",
            "createdAt": "2026-03-03T10:30:00.000Z",
            "updatedAt": "2026-03-03T10:30:00.000Z"
        }
    ],
    "total": 1,
    "limit": 20,
    "offset": 0
}
```
- Error Responses:
    - 401 Unauthorized (Not authenticated)
    - 403 Forbidden (Missing role: Admin/Office)

### 3. Get Customer Details
Fetches full details of a specific customer, including their linked projects

- Method: `GET`
- URL: `/:id`
- Projects contract:
    - `projects` is always returned as an array (empty array `[]` if no projects exist)
    - each project item includes exactly: `id`, `name`, `status`
- Success Response: 200 OK
- Response Body (example):
```json
{
    "id": "f7a9d6d0-0f31-4f7d-9a4e-9c6dc7f0e8e2",
    "customerCode": "C-000123",
    "type": "BUSINESS",
    "companyName": "Example Gardening Ltd",
    "firstName": null,
    "lastName": null,
    "phoneNumber": "+49 171 1234567",
    "email": "office@example-gardening.de",
    "streetHouseNumber": "Main Street 10",
    "postalCode": "60311",
    "city": "Frankfurt",
    "createdAt": "2026-03-03T10:30:00.000Z",
    "updatedAt": "2026-03-03T10:30:00.000Z",
    "projects": [
        {
            "id": "d6b21687-8d9e-4a4e-a9e9-30c8fe3bf0c7",
            "name": "Garden Redesign",
            "status": "ACTIVE"
        }
    ]
}
```
- Error Response: 404 Not Found
- Error Responses:
    - 401 Unauthorized (Not authenticated)
    - 403 Forbidden (Missing role: Admin/Office)
    - 404 Not Found (Customer does not exist)

### 4. Update Customer Data
Updates specific fields of an existing customer record

- Method: `PATCH`
- URL: `/:id`
- Request Body: Any subset of customer fields (optional)
- Validation: Same type-dependent rules as in `POST /` must be enforced.
- Update rules:
    - `customerCode` is read-only and cannot be changed
    - `type` can be changed (`PRIVATE` <-> `BUSINESS`)
    - if `type` is changed, required fields for the new type must be provided in the same request
    - empty request body `{}` is rejected with `400 Bad Request`
- Success Response: 200 OK
- Response Body (example):
```json
{
    "id": "f7a9d6d0-0f31-4f7d-9a4e-9c6dc7f0e8e2",
    "customerCode": "C-000123",
    "type": "BUSINESS",
    "companyName": "Example Gardening GmbH",
    "firstName": null,
    "lastName": null,
    "phoneNumber": "+49 171 1234567",
    "email": "office@example-gardening.de",
    "streetHouseNumber": "Main Street 12",
    "postalCode": "60311",
    "city": "Frankfurt",
    "createdAt": "2026-03-03T10:30:00.000Z",
    "updatedAt": "2026-03-03T11:10:00.000Z"
}
```
- Error Responses:
    - 400 Bad Request (Validation failed)
    - 401 Unauthorized (Not authenticated)
    - 403 Forbidden (Missing role: Admin/Office)
    - 404 Not Found (Customer does not exist)
    - 409 Conflict (Email already exists)

### Delete Customer
Removes a customer record 

- Method: `DELETE`
- URL: `/:id`
- Delete rule:
    - hard delete is blocked if the customer has linked records (e.g. projects, invoices)
    - in this case the API returns `409 Conflict`
- Success Response: 204 No Content
- Error Responses:
    - 401 Unauthorized (Not authenticated)
    - 403 Forbidden (Missing role: Admin/Office)
    - 404 Not Found (Customer does not exist)
    - 409 Conflict (Customer is still linked to business records)
